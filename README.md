# Project Description

Приложение позволяет пользователю вводить название города и получать актуальные метеорологические данные через публичное API (OpenWeatherMap). Реализовано кеширование последних данных для офлайн-доступа.

### Key Features
*   **Поиск погоды:** Ввод названия города с валидацией.
*   **Отображение данных:** Текущая температура, описание (ясно, дождь), иконка погоды, влажность, скорость ветра.
*   **Кеширование:** Сохранение последнего успешного запроса для просмотра без подключения к сети.
*   **Адаптивный UI:** Интерфейс подстраивается под платформенные гайдлайны (Material 3 на Android, плоский дизайн на iOS, минимализм на Linux, responsive grid на Web).
*   **Интернационализация:** Поддержка русского, английского и белорусского языков.

### Tech Stack
*   **UI:** Compose Multiplatform
*   **HTTP-клиент:** Ktor Client
*   **Сериализация:** kotlinx.serialization
*   **Асинхронность:** Kotlin Coroutines
*   **Локальное хранилище:** Multiplatform Settings / SQLDelight (для кеша)
*   **DI:** Koin (опционально)

---

## Platform-Specific UI Requirements

*   **Android:** Карточки Material 3 с тенями и закругленными углами; `TextField` с иконкой поиска.
*   **iOS:** Плоские карточки без теней; `SegmentedControl` для переключения между сохраненными городами; `SearchBar` вместо `TextField`.
*   **Linux:** Минималистичный дизайн, четкие границы элементов; возможность ресайза окна; текстовые поля с явными рамками.
*   **Web:** Отзывчивая верстка (CSS Grid/Layout): 1 колонка на мобильных, 2 на планшетах, 3 на десктопах.

## How to Build

Для сборки проектов необходимо:
* **Android Studio** (Hedgehog или новее) с плагином Kotlin Multiplatform.
* **Xcode** (для iOS-таргета).
* **JDK 17** или новее.

### Build Commands
*   **Android:** `./gradlew :composeApp:assembleDebug`
*   **iOS:** Открыть `iosApp` в Xcode и запустить сборку, либо `./gradlew :composeApp:iosSimulatorArm64Test`
*   **Desktop (Linux):** `./gradlew :composeApp:run`
*   **Web (Wasm/JS):** `./gradlew :composeApp:wasmJsBrowserRun`

## Testing

Проект содержит 3-7 тестов каждого вида, запускаемых через CI/CD.

*   **Модульные тесты:** Проверка логики маппинга данных, валидации ввода города, парсинга JSON.
*   **UI-тесты:** Проверка отображения карточек погоды, состояний загрузки и ошибок, адаптивной сетки (Web).
*   **Интеграционные тесты:** Сквозная проверка: ввод города -> HTTP-запрос -> отображение данных -> сохранение в кеш.

### Run Tests
`./gradlew allTests`
